## Lessons Learned

### attack-patterns (2 条)

- [high] 相同技术漏洞以"越权访问"提交补天（中危）比以"批量信息泄露"提交 edu（低危）评级高；同一目标优先选补天提交，标题聚焦"越权"而非"泄露规模"
- [medium] SaaS 平台多租户后端（统一 ID 不隔离）一旦突破一个漏洞即影响全部租户；单个越权 = 平台级危害，报告中应着重描述多租户影响面

### false-positives (2 条)

- [high] 补天对水卡类 App IDOR 主动降级：高危 → 中危——涉及学生个人信息但无直接财产损失，补天评级上限约中危；不要以为 55万用户 IDOR 能拿高危
- [high] 硬编码密钥（DES Key 泄露）补天判低危——纯密钥泄露不被补天视为高危，除非附带完整利用链证明实际财产/账号危害

### evidence-gaps (1 条)

- [medium] 0.3 元回报表明此类 SaaS 水卡目标性价比极低——技术复杂度高（APK 逆向+加固脱壳+加密分析）但平台评级天花板低，后续类似目标优先评估平台覆盖范围再决定是否投入
