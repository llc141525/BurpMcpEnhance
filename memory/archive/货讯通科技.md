# 货讯通科技(CargoSmart/OOCL/Weikayun) — 完结存档

归档时间: 2026-05-28

## 目标范围
- oocl.com / oocllogistics.com / weikayun.com / cargosmart.com / schedulingsmart.com

## Lessons Learned

### attack-patterns (3 条)

- [high] OTP 暴破类漏洞必须在 PoC 成功当天立即提交，接口修复极快（weikayun F-017/019/020 在提交后已加限频）；延迟提交等于无效
- [high] CORS 反射 + credentials:true 即使端点返回 401 也可收录（F-001/F-014 均通过）；核心证据是响应头而非响应体
- [medium] Keycloak Admin Console 公网可达（F-008 直接 200 返回完整 HTML）是强实际危害；此类管理面板无需额外 PoC 即可立案

### false-positives (4 条)

- [high] **补天不收手机号枚举**（sendVerificationCode 差异响应）— F-018 被明确告知"手机号枚举不收录"，以后直接放弃此类提交
- [high] 低危信息泄露（actuator/metrics、autodiscover.json、"Hello Koa 2!"默认页）被补天以"无实际危害"拒绝 — 需证明可利用性（提取凭证/PII/内网结构）才值得提交
- [high] WAF 完全拦截的端点（schedulingsmart F-040 返回 451）不可提交 — 无法复现即打回
- [medium] 已修复端点慎提交：F-002（moc.oocl.com）提交时已返回 403，虽有历史证据但风险较高

### evidence-gaps (3 条)

- [high] OTP 暴破提交时必须包含**完整成功截图**（验证码已被正确爆破并登录/重置成功）；仅凭 Burp Intruder 无限重试截图不足以证明
- [high] 信息泄露类漏洞需明确**危害链**：泄露内容 → 如何被利用 → 实际影响（补天评审要求"有意义的问题"）
- [medium] CORS 含 credentials:true 的漏洞建议附带一个简单 fetch PoC HTML，说明攻击者可跨域读取哪些接口数据

## 关键教训（给下个目标）

1. **速提交**：暴破/枚举类漏洞 PoC 成功后当天报告，不要等报告写完
2. **补天不要提手机号枚举**，永远放弃
3. **低危信息泄露**必须有"危害演示"段落，否则一律打回
4. **OTP 暴破报告**必须含成功截图，否则"无法复现"打回
